## 📝 基本使用流程

### 1. 首次使用设置

**步骤1**: 启动应用后，系统会自动加载默认配置的AI网站

**步骤2**: 点击左侧菜单中的"⚙️ 网站管理"，确认已启用的AI网站

**步骤3**: 返回"📋 提问对比"页面，开始提问

### 2. 提问对比操作

#### 简单提问
```
操作步骤:
1. 在问题输入框中输入问题
2. 点击"发送问题"按钮
3. 等待系统从多个AI网站获取回答
4. 对比查看不同AI的回答

示例问题:
- "什么是人工智能？"
- "如何学习Python编程？"
- "介绍几个提高工作效率的方法"
```

#### 快捷键操作
```
Enter        - 发送问题（单行模式）
Shift+Enter  - 换行（多行模式）
Ctrl+Enter   - 强制发送（多行模式）
```

## ⚙️ 高级功能使用

### 网站管理配置

#### 启用/禁用网站
```
操作路径: 左侧菜单 → ⚙️ 网站管理

步骤:
1. 查看网站列表和状态
2. 使用开关按钮启用/禁用特定网站
3. 点击"保存配置"应用更改

建议:
- 同时启用2-3个网站以获得最佳性能
- 需要登录的网站可以先禁用，待登录后再启用
```

#### 添加自定义网站
```
配置参数说明:
- 网站名称: 显示名称，如"ChatGPT"
- 网站URL: 完整的访问地址
- 回答选择器: 用于提取回答内容的CSS选择器
- 输入框选择器: 问题输入框的选择器
- 提交按钮选择器: 发送按钮的选择器
- 等待时间: 页面加载等待时间（毫秒）

示例配置:
网站名称: ChatGPT
网站URL: https://chat.openai.com
回答选择器: .markdown-content
输入框选择器: textarea[placeholder*="输入"]
提交按钮选择器: button[type="submit"]
等待时间: 3000
```

### API服务使用

#### 获取API密钥
```
操作路径: 左侧菜单 → 🔑 API配置

步骤:
1. 查看或生成API密钥
2. 配置服务端口（默认8080）
3. 启动本地API服务
4. 使用API密钥调用服务
```

#### API调用示例

**基本聊天请求:**
```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key-here" \
  -d '{
    "messages": [{"role": "user", "content": "什么是深度学习？"}],
    "model": "ai-comparison"
  }'
```

**流式响应:**
```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key-here" \
  -d '{
    "messages": [{"role": "user", "content": "解释一下量子计算"}],
    "model": "ai-comparison",
    "stream": true
  }'
```

## 💡 使用技巧

### 提问技巧

#### 获得更好的回答对比
```
✅ 好的提问方式:
- "详细解释机器学习的概念、应用和发展前景"
- "请从技术原理、应用场景和未来发展三个方面分析区块链技术"
- "对比传统编程和机器学习在解决问题上的差异"

❌ 避免的提问方式:
- "你好"（太简单，无法体现AI差异）
- "1+1=？"（标准答案，对比意义不大）
- 过于专业或冷门的问题（可能部分AI无法很好回答）
```

#### 问题类型建议
```
适合对比的问题类型:
📚 概念解释类: "解释XXX的概念和原理"
🔧 技术分析类: "分析XXX技术的优缺点"
💡 方案比较类: "比较不同解决方案的优劣"
🚀 趋势预测类: "预测XXX技术的发展趋势"
🛠️ 实现指导类: "如何实现XXX功能，有哪些方法"

不适合对比的问题类型:
❓ 简单事实类: "中国的首都是哪里"
🔢 数学计算类: "计算复杂的数学公式"
📅 实时信息类: "今天的股票价格"
```

### 性能优化建议

#### 提高响应速度
```
速度优化技巧:
1. 同时提问的网站数量控制在2-3个
2. 避免在网络状况差的时候使用
3. 关闭不需要的网站可以显著提高速度
4. 对于简单问题，响应会更快
```

#### 提高成功率
```
成功率优化技巧:
1. 确保网络连接稳定
2. 对于需要登录的网站，先完成登录
3. 适当延长等待时间配置
4. 遇到失败时可以重试发送
```

## 🛠️ 故障排除

### 常见问题解决

#### 问题: 某个网站总是获取失败
```
可能原因和解决方案:
1. 网络连接问题
   → 检查网络连接，尝试刷新页面
   
2. 网站需要登录
   → 在浏览器中完成登录，或禁用该网站
   
3. 选择器配置错误
   → 更新网站的选择器配置
   
4. 网站结构变化
   → 等待应用更新，或手动更新选择器
```

#### 问题: 响应速度很慢
```
优化方法:
1. 减少同时启用的网站数量
2. 检查网络带宽和延迟
3. 关闭其他占用网络的程序
4. 调整等待时间配置
```

#### 问题: 应用崩溃或无响应
```
解决方法:
1. 重启应用
2. 清理历史记录减少数据库负载
3. 检查系统资源使用情况
4. 查看日志文件了解详细错误
```
