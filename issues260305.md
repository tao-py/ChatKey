## 主要Issues

### 🔴 高优先级Issues

#### 1. 选择器匹配问题
**问题描述**: 多个测试用例中选择器无法正确匹配页面元素
**影响范围**: 功能流程测试、部分UI测试
**具体表现**:
- `getByRole('button', { name: 'xxx' })` 无法找到元素
- 页面元素文本内容与预期不符
- 严格模式下的元素匹配冲突

**建议修复**:
```typescript
// 修改前
page.getByRole('button', { name: 'API配置' })

// 修改后
page.getByRole('menuitem', { name: /API配置/ })
```

#### 2. 页面状态管理问题
**问题描述**: 输入框和按钮的禁用状态处理不当
**影响范围**: 提问功能测试
**具体表现**:
- 输入框默认被禁用，但测试预期为可编辑
- 提交按钮状态验证失败
- 需要先启用AI网站才能进行提问

**建议修复**:
- 更新测试逻辑，考虑实际的业务状态
- 在测试前先启用至少一个AI网站
- 验证禁用状态的合理性

#### 3. 异步操作超时
**问题描述**: 多个测试用例因异步操作超时而失败
**影响范围**: 功能流程测试
**具体表现**:
- `page.waitForLoadState('networkidle')` 超时
- 表单提交和页面切换超时
- 测试用例执行时间超过30秒限制

**建议修复**:
- 调整超时设置
- 优化异步操作等待策略
- 使用更智能的等待条件

### 🟡 中优先级Issues

#### 4. 测试数据准备不足
**问题描述**: 部分测试需要特定的前置数据状态
**影响范围**: 历史记录、网站管理功能测试
**具体表现**:
- 历史记录页面测试时无数据
- 网站管理测试时无网站数据
- 答案对比测试时无答案数据

**建议修复**:
- 在测试前准备必要的测试数据
- 使用mock数据或测试 fixtures
- 考虑数据库初始状态

#### 5. 跨浏览器兼容性问题
**问题描述**: 不同浏览器下测试结果不一致
**影响范围**: 多浏览器测试
**具体表现**:
- Chromium通过但Firefox失败
- 移动端浏览器特殊处理问题
- 响应式布局测试差异

**建议修复**:
- 针对不同浏览器优化选择器
- 调整移动端测试策略
- 考虑浏览器特性差异
